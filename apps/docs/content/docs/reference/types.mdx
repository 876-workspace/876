Every public type is exported from the package root and grouped here by domain:
[Common](#common) (client configuration, errors, result envelopes),
[Auth](#auth) (password and OTP flows, users, sessions), and [OAuth](#oauth)
(relying-party Sign in with 876 token exchange).

```ts
import type {
  // Common
  ClientOptions,
  SignInWith876Options,
  OAuthOptions,
  RequestOptions,
  AuthCredentials,
  AuthError,
  AuthErrorCode,
  Result,
  // Auth
  LoginParams,
  LoginResult,
  Session,
  User,
  AuthEvent,
  ListProvidersResult,
  // OAuth
  AuthorizationUrlParams,
  DiscoveryDocument,
  IntrospectParams,
  IntrospectResponse,
  OAuthError,
  OAuthResult,
  PkcePair,
  RefreshTokenParams,
  RevokeParams,
  RevokeResponse,
  TokenParams,
  TokenResponse,
  UserInfo,
  UserInfoParams,
} from '@876/sdk'
```

<Callout type="info">
  All app-owned timestamps (`createdAt`, `updatedAt` on app users) are **Unix
  epoch seconds**. WorkOS-boundary timestamps (`createdAt`, `updatedAt`,
  `lastSignInAt` on a WorkOS user) are **ISO 8601 strings**.
</Callout>

## Common

every auth method.

### AuthErrorCode

`AuthErrorCode` is a string because auth methods may pass through backend-only
API error codes that are not declared in the SDK-local registry.

```ts
type AuthErrorCode = string
```

Use `SdkAuthErrorCode` when calling SDK-local helpers such as
`createAuthError()`.

```ts
type SdkAuthErrorCode =
  | 'auth/client-not-configured'
  | 'auth/invalid-credentials'
  | 'auth/invalid-input'
  | 'auth/invalid-response'
  | 'auth/invalid-token'
  | 'auth/missing-code'
  | 'auth/missing-email'
  | 'auth/missing-first-name'
  | 'auth/missing-identifier'
  | 'auth/missing-last-name'
  | 'auth/missing-password'
  | 'auth/network-error'
```

### Result

The discriminated `{ data, error }` envelope returned by every auth method. On
success `error` is `null`; on failure `data` is `null`. See the
[result pattern](/docs/reference/result-pattern) for the recommended narrowing
idiom.

```ts
type Result<TSuccess> =
  | { data: TSuccess; error: null }
  | { data: null; error: AuthError }
```

### SDK876Client

The namespaced client returned by [`create876Client()`](/docs/auth/create-client).
`auth` holds the implemented [Auth](#auth) methods; `users` exposes profile,
addresses, and contacts sub-resources; `oauth` provides OIDC relying-party
operations when configured.

```ts
type SDK876Client = ReturnType<typeof create876Client>
```

### SDK876AuthClient

The `auth` namespace of [`SDK876Client`](#sdk876client) — the object exposing
`login`, `register`, `getSession`, and the rest. Used as the `client` value by
the embeddable auth UI.

```ts
type SDK876AuthClient = SDK876Client['auth']
```

## Auth

Method parameters, responses, users, sessions, and events from `types/auth.ts`.
Each method's `*Result` type is `Result<*Response>`.

### user

user** (consumer / enterprise accounts the platform owns) and a **WorkOS user**
(the raw WorkOS boundary user). Both carry `object: 'user'`; distinguish them by
the presence of `accountType` (app user only).

```ts
type User = AppUser | WorkosUser
```

### Method responses

The success payload (`data`) for each method.

```ts
// login() / register() / registerBusiness() — a session or a pending event
type LoginResponse = Session | AuthEvent
type RegisterResponse = Session | AuthEvent

// socialLogin() — a redirect to the provider
type SocialLoginResponse = Redirect
type Redirect = { url: string }

// verifyEmailCode() / verifyMagicOtp() — the authenticated user
type VerifyEmailCodeResponse = { user: User }
type VerifyMagicOtpResponse = { user: User }

// recover() / resetPassword() — a confirmation echoing the email
type RecoverResponse = PasswordRecovery
type ResetPasswordResponse = PasswordReset
type PasswordRecovery = { email: string }
type PasswordReset = { email: string }

// logout() — an empty confirmation object
type LogoutResponse = Logout
type Logout = Record<string, never>

// getSession() — the active session
type GetSessionResponse = Session

// resolve() — what auth methods are available for an identifier
type ResolveResponse = {
  email: string
  exists?: boolean
  business?: boolean
  methods?: AuthMethod[]
  ssoProvider?: SocialProvider | null
}

// sendMagicOtp() — when the code can next be re-sent
type SendMagicOtpResponse = { email: string; canResendAt: number }
```

### Method results

Every `*Result` is the [`Result`](#result) envelope wrapping the matching
response.

```ts
type LoginResult = Result<LoginResponse>
type RegisterResult = Result<RegisterResponse>
type RegisterBusinessResult = RegisterResult
type SocialLoginResult = Result<SocialLoginResponse>
type VerifyEmailCodeResult = Result<VerifyEmailCodeResponse>
type VerifyMagicOtpResult = Result<VerifyMagicOtpResponse>
type RecoverResult = Result<RecoverResponse>
type ResetPasswordResult = Result<ResetPasswordResponse>
type LogoutResult = Result<LogoutResponse>
type GetSessionResult = Result<GetSessionResponse>
type ResolveResult = Result<ResolveResponse>
type SendMagicOtpResult = Result<SendMagicOtpResponse>
```
